    Get started

    Install Needle, point it at your project, and connect it to your AI tool. Takes about 5 minutes.

    1 — Install

    Download the binary for your system. No build tools required.

    1

    Download

    Get needle-windows-x64.exe from the releases page, rename it to needle.exe.

    2

    Put it on your PATH

    Move the file somewhere Windows can find it, like C:\Windows\. Or add its folder to your PATH in System Settings.

    3

    Check it works

    needle --version
    1

    Download

    Get the right binary from the releases page:
    needle-macos-arm64 — M1/M2/M3 Mac
    needle-macos-x64 — older Intel Mac

    2

    Make it runnable and move it to PATH

    chmod +x needle-macos-arm64
sudo mv needle-macos-arm64 /usr/local/bin/needle

    If macOS blocks it ("unverified developer"), go to System Settings → Privacy & Security → click Allow.

    3

    Check it works

    needle --version
    1

    Download & install

    Get needle-linux-x64 from the releases page. It's a static binary — no dependencies needed.

    chmod +x needle-linux-x64
sudo mv needle-linux-x64 /usr/local/bin/needle
    2

    Check it works

    needle --version
    You'll need Rust installed

    If you don't have it: curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh — takes about 2 minutes.

    1

    Clone and install

    git clone https://github.com/your-org/needle
cd needle
cargo install --path .

    The binary lands at ~/.cargo/bin/needle which is already on your PATH after installing Rust.

    2 — Index your project

    Point Needle at a folder. It reads every file, builds a searchable index, and saves it locally. Nothing leaves your machine.

    1

    Run needle init

    needle init ./my-project

    Takes 10–60 seconds depending on project size. You'll see a progress bar while it works.

    2

    Start searching

    needle serve

    Opens this UI at http://localhost:7700. Keep this running while you work.

    3

    Keep the index up to date (optional)

    Run this in a separate terminal to auto-reindex whenever you save a file:

    needle watch

    3 — Connect your AI coding tool

    Once connected, your AI can search your codebase on its own — no more pasting code into the chat or telling it which files to look at. Pick your tool:

    After this, Claude can run search_code, find_callers, get_endpoints and 4 other tools against your local index automatically.
    1

    Run this once in your terminal

    claude mcp add needle needle mcp
    2

    Restart Claude Code

    That's it. Try asking: "search for authentication logic" — Claude will call Needle directly instead of guessing.

    Cursor's AI reads your index when it needs to find code — instead of loading entire files into its context window.
    1

    Create this file in your project root

    File: .cursor/mcp.json

    {
  "mcpServers": {
    "needle": {
      "command": "needle",
      "args": ["mcp"]
    }
  }
}
    2

    Restart Cursor

    Cursor picks up .cursor/mcp.json automatically. You'll see Needle listed under MCP tools in Settings → Features → MCP.

    Windsurf's Cascade can call Needle when it wants to find functions, callers, or endpoints in your project.
    1

    Open Windsurf's MCP config file

    File: ~/.codeium/windsurf/mcp_config.json (create it if it doesn't exist)

    {
  "mcpServers": {
    "needle": {
      "command": "needle",
      "args": ["mcp"]
    }
  }
}
    2

    Restart Windsurf

    Needle will appear in the MCP tools list. Cascade will use it automatically when you ask about your codebase.

    GitHub Copilot Chat in VS Code can call Needle tools in agent mode — searching your local index instead of guessing from file names.
    1

    Create this file in your project root

    File: .vscode/mcp.json

    {
  "servers": {
    "needle": {
      "type": "stdio",
      "command": "needle",
      "args": ["mcp"]
    }
  }
}
    2

    Enable agent mode in Copilot Chat

    Open Copilot Chat → switch to Agent mode → Needle tools appear automatically.

    Google's Gemini CLI supports MCP tools. Add Needle so Gemini can search your codebase directly from the terminal.
    1

    Edit your Gemini CLI settings

    File: ~/.gemini/settings.json

    {
  "mcpServers": {
    "needle": {
      "command": "needle",
      "args": ["mcp"]
    }
  }
}
    2

    Start a new Gemini session

    Run gemini — Needle tools load automatically. Try: "search for the login function".

    OpenAI's Codex CLI picks up MCP servers from its config file. Needle gives it fast, local codebase search.
    1

    Edit your Codex config

    File: ~/.codex/config.yaml (create if it doesn't exist)

    mcpServers:
  needle:
    command: needle
    args: [mcp]
    2

    Start Codex

    Run codex in your project folder. Needle's search tools will be available automatically.

    Continue.dev's AI assistant can call Needle to search your codebase without loading files into context.
    1

    Edit your Continue config

    File: ~/.continue/config.json

    Add this inside the top-level object:

    "experimental": {
  "modelContextProtocolServers": [
    {
      "transport": {
        "type": "stdio",
        "command": "needle",
        "args": ["mcp"]
      }
    }
  ]
}
    2

    Reload Continue

    Open the Continue panel → click the reload icon. Needle tools appear under available context providers.

    Any tool that supports the MCP standard (stdio transport) can connect to Needle.
    1

    Find your tool's MCP config

    Look for a setting called MCP servers, tools, or integrations. Add this entry:

    {
  "name": "needle",
  "command": "needle",
  "args": ["mcp"],
  "type": "stdio"
}
    2

    What Needle exposes

    Your tool gets 7 tools: search_code, find_callers, find_callees, get_endpoints, get_file_structure, find_similar, get_stats.

    Make sure needle serve is running

    The AI connection works by launching needle mcp in the background. Needle reads the index from ~/.needle/index/ — so you need to have run needle init first.

    Privacy Policy

    Effective date: June 2025 · Needle is a local-only tool.

    The short version

    Needle collects nothing. Your code never leaves your machine. There are no analytics, no telemetry, no accounts, and no servers controlled by the Needle project that your data touches.

    What data does Needle access?

    When you run needle init, Needle reads the source files in the directories you specify. It:

    • Chunks each file into smaller pieces
    • Generates embeddings for each chunk (locally, via Ollama or hash-projection)
    • Stores the chunks, embeddings, and inverted index in a local directory on your disk (default: ~/.needle/index/)

    None of this data is transmitted anywhere.

    Ollama (optional)

    If you have Ollama running locally, Needle sends text to the local Ollama API at http://localhost:11434 to generate embeddings and LLM responses. This is a local HTTP call — data does not leave your machine.

    Groq (optional fallback)

    If you set a GROQ_API_KEY environment variable and Ollama is unavailable, the Ask feature will send your question and retrieved code chunks to the Groq API. In this case, that data is subject to Groq's privacy policy. This fallback is optional and requires you to explicitly provide an API key.

    Browser storage

    The Needle web UI stores your preferences (theme, search defaults, Ollama URL) in your browser's localStorage. This data is stored locally on your device and is not transmitted anywhere.

    Telemetry

    Needle collects no usage analytics, crash reports, or telemetry of any kind. There is no tracking code in the binary or the UI.

    Needle is an open-source project. If you have questions, open an issue in the repository.
